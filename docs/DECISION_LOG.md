# DECISION LOG（架构决策记录）

## ADR-001：保持 ID 稳定优先
- 日期：2026-02-07
- 决策：地图/NPC/事件/敌人 ID 默认不可变
- 原因：兼容旧档与历史事件引用
- 影响：扩展时优先新增而非重命名

## ADR-002：事件系统继续数据驱动
- 日期：2026-02-07
- 决策：条件与效果扩展到 schema，不在场景层硬编码大量剧情逻辑
- 原因：新 AI 接手成本更低，行为更可追踪
- 影响：`events.js` 规模会增大，后续应拆分任务文件

## ADR-003：先文档约束，再模块拆分
- 日期：2026-02-07
- 决策：先建立完整接手文档和流程，再做大型代码拆分
- 原因：当前工作流是多 AI 接力，交接稳定性优先
- 影响：近期会新增文档维护成本，但长期返工减少
